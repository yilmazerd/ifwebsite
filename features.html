Current Functionality and Limitations 

As of today, Instant Function is Java only. Python will be added next.


Execution for Java Code
Basic execution of Java programs are around feature "Launch Single-File Source-Code Programs" introduced in JDK 11 release. A single-file program is one where the program fits in a single source file. The JVM compiles the source file into memory and then runs the public main() method.
You can test your code before submitting by typing "Java <myCodeName.java>" in your terminal. This limits the libraries that can be imported but since our scope is to run simple functinoality in the API, that has not been an issue.

Response Code
Essentially the program will return what is being logged. You can customize the http response code. In order to do this, you need to print the following statement
System.out.println("httpStatusCode:"+httpStatusCode);
if this is not printed, then it will return 200.. 

Response Structure
If you customize the response code then we look for a log statement in the following structure
System.out.println("responseObject:"+responseObject);
once found, we return response object as JSON (at least we try). Response format is JSON and currently can not be edited. If you have a use case where you need to return non-json, shoot an email with your use case.
